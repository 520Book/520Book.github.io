# 使用

# 从零开始新建仓库

比如我们在D盘新建一个任意名字目录，然后在命令行进入该目录，输入以下命令：

git init
1

这个目录是空的，或者是现有存在的，都可以。

命令完成后，Git就在当前目录创建了一个空的仓库（empty Git repository），并会在当前目录下自动创建一个.git的目录，这个目录是Git来跟踪管理版本库的，千万不要手动修改这个目录里面的文件。

如果你使用Windows系统，为了避免遇到各种莫名其妙的问题，请确保目录名（包括父目录）不包含中文。

# 编辑文件并加入版本库

现在建立了仓库，我们就可以开始编辑文件了。随便添加一个文件，比如test.txt，然后准备把它提交到版本库。记住，提交分为两个步骤：首先要把文件添加到仓库，命令格式为：git add <文件名字>，下面我们就可以运行命令：

git add test.txt
1

这样就将test.txt添加到仓库了。如果仅仅添加了一个新文件我们可以这样解决，如果文件多了怎么办？不可能一个个去添加，对了，这时候我们可以使用神奇的命令来解决：

git add .
1

这个小点就是代表当前目录下全部的文件！

然后设置用户名和邮箱地址。每个机器都必须自报家门：你的名字和Email地址。

git config --global user.name "test"
git config --global user.email "test@gmail.com"
1
2

注意： global参数表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和Email地址。

第二步：将文件提交到仓库，命令格式为git commit -m <本次提交的说明>。下面我们就可以输入命令将test.txt添加到仓库了。

git commit -m first
1

提醒：这个说明，仅仅是个说明，也就说，你每次用同一个名字的说明不会有任何问题。

# 将本地仓库上传到远程仓库

虽然我们已经在GitHub建立了仓库，但是Git并不知道，那么如何让Git知道我们有了一个远程仓库呢？这个时候就需要在Git中先添加远程仓库，命令格式为：git remote add [shortname] [url]。shortname就是别名。下面我们运行命令：

git remote add test https://github.com/Heart-King/test.git
1

运行后我们可以通过不加参数的git remote命令查看当前的远程库。现在，我们就可以把刚才已经建立的本地仓库文件，推送到远程仓库了。

DANGER

需要记住的一点是，所谓的远程库，也仅仅是当前仓库的远程库！也就是说你在其他目录建立一个新的仓库后，它是没有这个远程库的，你需要另外添加。

git push test main
1

test是远程仓库的名字，main是我们本地仓库的名字。

这个时候会提示我们输入GitHub的账号和密码，按提示输入即可。

test我们知道，使我们自己刚才取的，那这个main名字是哪里来的呢？其实这是系统默认给我们取的。可以使用如下命令查看：

git branch
1

通常我们的主要分支名字默认都是叫main，如果我们要修改分支名字，我们输入以下命令给它重命名：

git branch -m main adong
1

这里需要注意的一点就是：我们给分支修改名字后，再推送到远程仓库的时候，远程仓库也会创建一个新的叫adong的分支！！！

# 利用SSH通信

在本地电脑生成SSH Key之后，配置到github里，就不用像http协议那样每次都输入账户密码了。

  • 第一步：检查是否已经存在，有的话可以跳过第二步，直接拷贝
ls -al ~/.ssh
1
  • 第二步：生成一个Key
ssh-keygen -t rsa -C "your_email@example.com"
1

TIP

这里的邮件地址your_email@example.com可以任意取名，和你的GitHub登录账户的Email地址没有关系。

  • 第三步：将Key拷贝到剪切板
# sudo apt-get install xclip # xclip是linux的拷贝插件
xclip -sel clip < ~/.ssh/id_rsa.pub
1
2
pbcopy < ~/.ssh/id_rsa.pub
1
clip < ~/.ssh/id_rsa.pub
1

然后打开Github的账户设置，找到SSH and GPG keys，然后新建一个SSH key，直接粘贴进去就大功告成啦！

git branch -a

git branch -u origin/master master
git branch --set-upstream-to=origin/master master

git branch -m newName
git branch -M newName

# 删除分支，使用-D强制删除。
git branch -d branchname

# 删除远程分支
git branch -d -r branchname

# .gitignore规则不生效

.gitignore只能忽略那些原来没有被track的文件，如果某些文件已经被纳入了版本管理中，则修改.gitignore是无效的。解决方法就是先把本地缓存删除（改变成未track状态），然后再提交:

git rm -r --cached .
git add .
git commit -m "update .gitignore"
1
2
3

# 常见问题

# 关于回车符的问题

由于Dos和Windows平台使用CR+LF来结束一行，而Mac和 Linux平台：只使用换行LF一个字符来结束一行，导致两个平台的不一致，Git会出警告。可以用以下命令在提交时转换为LF，检出时转换为CRLF。

git config --global core.autocrlf true
1

# 拒绝合并分支

如果提示fatal:refusing to merge unrelated histories，原因在于基于同一个仓库，做了两次完全不同的提交。比如一次提交是在github上直接创建文件并commit，而第二次是在本地还没有拉新代码的情况下修改了代码，并进行pull -> commit 和push。解决很简单，添加一个--allow-unrelated-histories参数即可：

$ git pull test main --allow-unrelated-histories
1

# 关于主机授权

如果提示The authenticity of host 'github.com (20.205.243.166)' can't be established，我们可以查看自己用户目录下的.ssh目录下是否有三个密钥文件。若只有两个的话，输入yes回车之后，会生成一个known_hosts文件即可。
