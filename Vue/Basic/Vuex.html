Vuex官网open in new window

每一个Vuex应用的核心就是store。store就是一个容器，它包含着你的应用中大部分的状态state，或者理解为全局的数据。但是，Vuex和单纯的全局对象有以下两点不同：

  • Vuex的状态存储是响应式的。当Vue组件从store中读取状态的时候，若store中的状态发生变化，那么相应的组件也会相应地得到高效更新。
  • 你不能直接改变store中的状态。改变store中的状态的唯一途径就是显式地提交commit mutation。

# 安装Vuex

npm i vuex -S

# 简单的实例

在main.js中加入下面代码：

import Vuex from "vuex";
Vue.use(Vuex);

const store = new Vuex.Store({
  state: {
    str: "Hello",
  },
  mutations: {
    sayHello(state, name) {
      state.str += " " + name;
    },
  },
});

console.log(store.state.str);
store.commit("sayHello", "adong");
console.log(store.state.str);
store.state.str = "SB vue";
console.log(store.state.str);

注意，每个mutation都有一个字符串的事件类型(type)和 一个回调函数 handler。这个回调函数就是我们实际进行状态更改的地方，并且它会接受state作为第一个参数。

通过这个实例我们大体可以明白store到底是干嘛的，说白了就是个全局对象罢了。store.state 来获取状态对象，以及通过store.commit方法触发状态变更：那为什么简单的事情要复杂化？主要是为了响应式。

# 在组件中获取store

上面的例子我们直接在main.js中操作，那如何在其它组件中引用store呢？首先，我们在根vue实例化中要加入store，比如在main.js中加入如下代码：

new Vue({
  store, // 将store加入，这个store就是我们前面定义的
  render: (h) => h(App),
}).$mount("#app");

然后我们在子组件中的computed属性中就可以通过this.$store访问到store了。

// view
<label>{{ str }}</label>

// js
computed: {
  str () {
    return this.$store.state.str
  }
}

# mapState 辅助函数

当一个组件需要获取多个状态时候，可以使用mapState辅助函数帮助我们生成计算属性。

import { mapState } from "vuex";

computed:
mapState({
  // 箭头函数可使代码更简练
  str: (state) => state.str,
});

当映射的计算属性的名称与state的子节点名称相同时，我们也可以给mapState传一个字符串数组来进一步简化。

computed:
mapState([
  "str",
]);

如果还有其它本地属性需要计算，我们可以使用...来展开mapState，比如：

computed: {
  localComputed () { /* ... */ },

  // 使用对象展开运算符将此对象混入到外部对象中
  ...mapState([
    'str'
  ])
}

Action与Mutation的唯一区别就在于，Mutation里面只支持同步操作，而Action可以支持异步操作。并且，很重要的一点，Action是依赖于Mutation而生存的！！
